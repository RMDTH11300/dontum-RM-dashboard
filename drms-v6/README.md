# DRMS Sprint 5 — Risk Register อัตโนมัติ

เพิ่มการสร้าง Risk Register จาก Incident และ Risk Profile ตามหน่วยงานหลักที่แก้ไข พร้อมส่งออก CSV และพิมพ์ PDF

## จุดสำคัญ
- ไม่กระทบ Dashboard V1
- ปีงบประมาณยึดจากชุดข้อมูลโดยตรง
- เลือกหลายหน่วยงานได้
- มาตรการและการติดตามเป็นร่างตั้งต้น ต้องให้หน่วยงานเจ้าภาพทบทวนก่อนใช้จริง


## Sprint 6
- Risk Matrix 5×5 จาก Risk Profile อัตโนมัติ
- แนวโน้มเปรียบเทียบปีงบประมาณ 2566–2569
- Checklist ความพร้อมข้อมูลสำหรับ HA
- สรุปข้อความสำหรับการทบทวนคุณภาพในรายงาน HA


## Sprint 7
- เพิ่มมุมมองสรุปและแบบฟอร์ม HA สำหรับ Risk Profile / Risk Register
- ช่องวัตถุประสงค์ ตัวชี้วัด เป้าหมาย มาตรการเพิ่มเติม ผู้รับผิดชอบ กำหนดเวลา และผลติดตาม แก้ไขได้
- บันทึกร่างในเบราว์เซอร์ของเครื่องผู้ใช้
- แสดงประวัติการบันทึกร่าง
- ส่งออก Word, Excel, CSV และพิมพ์/PDF


## Sprint 8
- เพิ่ม HA Report Generator แบบแก้ไขข้อความได้
- เพิ่มหัวข้อบริบท จุดเด่น ประเด็นพัฒนา แผนพัฒนา และ Checklist หลักฐาน
- บันทึกร่างรายงาน HA ในเบราว์เซอร์
- เก็บประวัติการบันทึก
- ส่งออก Word, Excel, CSV และ PDF


## DRMS V6.1 Final
หน้า Admin เปลี่ยนเป็น Data Center สำหรับ:
- นำเข้า Excel จากระบบเดิม
- ตรวจสอบ Data Quality
- ส่งออก JSON สำหรับ GitHub
- สำรอง/กู้คืนข้อมูล
- ตั้งค่าระบบ
- บันทึกประวัติการนำเข้าและส่งออก

หมายเหตุ: ปีงบประมาณถูกกำหนดจากช่องที่ Admin เลือกโดยตรง ส่วนวันที่ใช้เพื่อตรวจสอบและแจ้งเตือนเท่านั้น ระบบจะไม่เปลี่ยนปีงบประมาณของชุดข้อมูลเอง


## DRMS V6.2
- กรองตัวอย่างข้อมูลตามประเภทปัญหา Data Quality
- เลือกตัดข้อมูลซ้ำ ข้อมูลนอกช่วงปีงบประมาณ ข้อมูลไม่มีรหัส และข้อมูลไม่มีหน่วยงานหลัก เฉพาะตอนส่งออก
- แสดงจำนวนรายการที่จะส่งออกจริง
- ส่งออก Incident JSON และไฟล์ metadata สำหรับตรวจสอบยอด Dashboard
- ไม่แก้ไขข้อมูลต้นฉบับที่นำเข้า


## V6.3 – โครงสร้างหน่วยงานแบบลำดับชั้น
ตัวกรองหน่วยงานปรับเป็น 3 ระดับ:
- กลุ่มงาน
- งาน
- งานย่อย

รองรับการเลือกทั้งกลุ่มงาน ทั้งงาน หรือเลือกเฉพาะงานย่อย และเลือกหลายกลุ่มพร้อมกันได้  
Master หน่วยงานอยู่ที่ `config/departments.json`


## V6.4 – RCA Manager
- กรองเฉพาะ Incident ระดับ E–I และ 3–5
- หน้า Admin แสดงมี/ไม่มี RCA และแนบ PDF/DOC/DOCX
- อัปโหลดไฟล์เข้า GitHub ผ่าน GitHub API
- Token ไม่ถูกฝังในโปรแกรม และเก็บเฉพาะ Session Storage
- อัปเดต `data/rca.json` อัตโนมัติ
- หน้า Public แสดงมี/ไม่มี RCA และปุ่มดาวน์โหลด


## V6.5 – RCA Easy Upload
- ไม่ใช้ GitHub Token
- Admin เลือก Incident และไฟล์ RCA
- ระบบเปลี่ยนชื่อไฟล์ตาม Incident อัตโนมัติ
- ดาวน์โหลดไฟล์ RCA ที่พร้อมอัปโหลด
- ดาวน์โหลด `rca.json` ฉบับอัปเดต
- ผู้ดูแลอัปโหลดไฟล์ RCA ไปที่ `drms-v6/rca/ปี/`
- ผู้ดูแลอัปโหลด `rca.json` ไปที่โฟลเดอร์ `data` กลาง


## V6.6 – RCA Manager ฉบับสมบูรณ์
- ตัด GitHub Token และการเชื่อมต่อ API ออกทั้งหมด
- แสดงเฉพาะ Incident ระดับ E–I และ 3–5
- แสดง “มี RCA / ไม่มี RCA”
- เลือก PDF/DOC/DOCX แล้วเปลี่ยนชื่อเป็นเลข Incident อัตโนมัติ
- เก็บรายการไฟล์ที่เตรียมไว้ในเบราว์เซอร์
- เตรียมหลายไฟล์ก่อนดาวน์โหลด `rca.json` ได้
- มีปุ่มดาวน์โหลด `rca.json` แยกชัดเจน
- มีตำแหน่งอัปโหลด GitHub แสดงในหน้า Admin
- หน้า Public แสดงปุ่มดาวน์โหลด RCA เมื่อเผยแพร่แล้ว
