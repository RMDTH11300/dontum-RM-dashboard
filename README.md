# Dashboard ระบบบริหารความเสี่ยง โรงพยาบาลดอนตูม

Dashboard แบบอ่านอย่างเดียวสำหรับเผยแพร่บน GitHub Pages โดยใช้ข้อมูลอุบัติการณ์, Risk Profile และ Risk Register ปีงบประมาณ 2566–2569

## จุดสำคัญ: แก้ปัญหาปี พ.ศ. กับปีงบประมาณแล้ว

ระบบ **ไม่คำนวณปีงบประมาณจากวันที่เกิดเหตุ** แต่กำหนดปีงบประมาณจากชุดไฟล์ต้นทางโดยตรง

- ไฟล์ `66_...xlsx` → ปีงบประมาณ 2566 ทุกแถว
- ไฟล์ `67_...xlsx` → ปีงบประมาณ 2567 ทุกแถว
- ไฟล์ `68_...xlsx` → ปีงบประมาณ 2568 ทุกแถว
- ไฟล์ `69_...xlsx` → ปีงบประมาณ 2569 ทุกแถว

วันที่เกิดเหตุใช้เฉพาะสำหรับแสดงวันที่และจัดลำดับเดือน ต.ค.–ก.ย. เท่านั้น

## ความสามารถ

- เลือกดูข้อมูลแยกปีงบประมาณ หรือภาพรวมทุกปี
- KPI จำนวนอุบัติการณ์ Clinical, Non-clinical และระดับรุนแรงสูง
- กราฟแนวโน้มรายปี รายเดือน ระดับความรุนแรง รหัสความเสี่ยง และหน่วยงาน
- ค้นหาข้อมูลจากทุกช่องในรายการอุบัติการณ์
- กรองประเภท ความรุนแรง หน่วยงาน และเดือนงบประมาณ
- เปิดรายละเอียดอุบัติการณ์รายเคส
- ส่งออกผลการค้นหาเป็น CSV
- ดู Risk Profile และ Risk Register แยกปี
- ใช้งานได้บนมือถือและคอมพิวเตอร์

## นำขึ้น GitHub Pages

1. แตกไฟล์ ZIP
2. อัปโหลด **ไฟล์และโฟลเดอร์ทั้งหมดภายใน** ไปที่หน้าแรกของ Repository โดยต้องเห็น `index.html`, `assets`, `data` และ `tools`
3. ไปที่ **Settings → Pages**
4. เลือก **Deploy from a branch**
5. เลือก Branch `main` และ Folder `/ (root)` แล้วกด Save
6. เปิดลิงก์ GitHub Pages ที่ระบบแสดง

อย่าอัปโหลดเฉพาะ `index.html` เพราะข้อมูลอยู่ในโฟลเดอร์ `data`

## ทดลองเปิดในเครื่อง

การดับเบิลคลิก `index.html` อาจโหลด JSON ไม่ได้จากข้อจำกัดของเบราว์เซอร์ ให้เปิด Command Prompt ในโฟลเดอร์นี้แล้วใช้

```bash
python -m http.server 8000
```

จากนั้นเปิด `http://localhost:8000`

## อัปเดตข้อมูลในอนาคต

1. สร้างโฟลเดอร์ `source` แล้ววางไฟล์ Excel ชุดใหม่ลงไป โดยคงปีไว้ในชื่อไฟล์
2. เปิด Command Prompt ที่โฟลเดอร์เว็บไซต์
3. รันคำสั่ง

```bash
python tools/build_data.py --source source --output .
```

สคริปต์จะสร้างข้อมูลใน `data` ใหม่ โดยยึดปีงบประมาณจากชื่อไฟล์เช่นเดิม

## คำเตือนเรื่องความลับของข้อมูล

GitHub Pages เป็นเว็บไซต์สาธารณะและไม่มีระบบล็อกอินที่ปลอดภัย ไฟล์ JSON ในโฟลเดอร์ `data` สามารถถูกดาวน์โหลดได้ทั้งหมด ควรลบข้อมูลที่ระบุตัวบุคคล ข้อมูลผู้ป่วย หรือรายละเอียดอ่อนไหวก่อนเผยแพร่ หากต้องการระบบภายในที่มีผู้ใช้/รหัสผ่าน ควรติดตั้งบนเซิร์ฟเวอร์ที่มี Backend และระบบยืนยันตัวตน
