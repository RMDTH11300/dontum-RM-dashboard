# DRMS Data Dictionary

ฐานข้อมูล Incident ใช้โครงสร้างเดียวกับ `data/meta.json` ของ Dashboard เดิม

หลักสำคัญ
- ปีงบประมาณกำหนดจากไฟล์ `incidents_XXXX.json` โดยตรง ไม่คำนวณจากวันที่
- Risk Profile และ Risk Register ใช้ `กลุ่ม/หน่วยงานหลัก` เป็นเจ้าภาพ
- ช่องสถานะไม่แสดงในระบบ Public
- ระบบรองรับการเลือกหลายหน่วยงานพร้อมกัน

ข้อมูลสำคัญที่ใช้
1. รหัสรายงานอุบัติการณ์
2. รหัส: เรื่องอุบัติการณ์
3. เรื่องย่อย
4. ความรุนแรง
5. หน่วยงานที่บันทึกรายงาน
6. สถานที่เกิดเหตุ
7. รายละเอียดการเกิด
8. การจัดการเบื้องต้น
9. ข้อเสนอแนะเพื่อการแก้ไข
10. กลุ่ม/หน่วยงานหลัก
11. วันที่เกิดอุบัติการณ์
